# 🏗️ Implementation Plan: Local LLM Test Generator

**Blueprint ID:** LLM-TG-V0.1
**Status:** In Progress
**Framework:** React (Frontend) + Express (Backend)

---

## 🏗️ Architecture Overview

The system follows a 3-layer architecture separation:
1. **Frontend (Layer 3 - UI):** React + TailwindCSS.
2. **Backend (Layer 2 - Orchestration):** Express + Node.js (TypeScript).
3. **Storage (Layer 1 - Data):** SQLite using `sqlite3` for configuration and history.

---

## 🛠️ Data Strategy
- **Configuration:** Stored in `Settings` table.
- **History:** Stored in `TestGenerations` table.
- **Payload:** Strict Markdown format generated by LLMs following the `systemPrompt` defined in `llmController.ts`.

---

## 🧩 Components (Frontend)
| Component | Responsibility |
|---|---|
| `Sidebar` | List previous test case generations (History). |
| `ChatRoom` | Display generated test cases in Markdown format. |
| `InputArea` | User interface for entering Jira requirements and selecting a provider. |
| `SettingsPanel` | CRUD operations for LLM providers (API Keys, Endpoints). |

---

## ⚙️ API Endpoints (Backend)
| Method | Endpoint | Description |
|---|---|---|
| `GET` | `/api/settings` | Retrieve all LLM provider settings. |
| `POST` | `/api/settings` | Save/Update provider settings. |
| `POST` | `/api/generate` | Trigger LLM generation for a requirement. |
| `GET` | `/api/history` | Retrieve previous generations. |
